UX Laws & Principles
select all that apply
🧠
Each selected law gets encoded into the generated prompt as an explicit implementation directive for Cowork. Cowork will apply the law at component, layout, and interaction level.
Perception & Cognition
Fitts's Law
Interaction · Targeting
Time to reach a target is a function of its size and distance. Larger, closer targets are faster to click.
→ Primary CTAs min 44×44px. Sticky actions at thumb reach on mobile. Destructive actions small and distant.
Hick's Law
Decision Making
Decision time increases logarithmically with the number of choices. Fewer options = faster decisions.
→ Max 5–7 nav items. Progressive disclosure for advanced options. Wizard patterns for complex flows.
Miller's Law
Working Memory
The average person can hold 7 ± 2 items in working memory at once.
→ Chunk related items into groups of 5–9. Break long forms into steps. Group settings into categories.
Jakob's Law
Familiarity
Users spend most of their time on other sites. They prefer your site to work like sites they already know.
→ Use familiar UI patterns (header nav, modal overlays, breadcrumbs). Innovate on content, not convention.
Gestalt Principles
Visual Perception
Proximity, similarity, continuity, closure, and figure-ground. The brain groups visual elements into wholes.
→ Use spacing to group related items (proximity). Consistent size/colour for similar elements. White space as structure.
Serial Position Effect
Memory · Attention
Users best remember first and last items in a list. Middle items are recalled least (primacy + recency effects).
→ Place most important nav items first and last. Key CTAs at bottom of sections. Summary at end of forms.
Attention & Engagement
Zeigarnik Effect
Incomplete Tasks
People remember uncompleted tasks better than completed ones. Partial progress creates cognitive tension that drives action.
→ Progress bars, onboarding checklists, profile completion %. Show "85% complete" to pull users back.
Von Restorff Effect
Isolation Effect
An item that stands out from its peers is more likely to be remembered. Distinctiveness creates attention.
→ Highlight recommended plan with accent border. Primary CTA uses brand colour. Badge new features distinctly.
Peak-End Rule
Experience Memory
People judge an experience by its most intense moment (peak) and how it ends — not the average.
→ Design delightful success states. Celebrate form completions. Make onboarding's last step feel like an achievement.
Doherty Threshold
Response Time
Productivity soars when a computer and user interact at a pace ≤400ms per interaction. Delays destroy engagement.
→ Optimistic UI updates. Skeleton loading states. Target <100ms perceived response for all interactions.
Fogg Behaviour Model
Motivation + Ability
Behaviour = Motivation × Ability × Prompt. All three must align for a behaviour to occur.
→ Reduce ability barriers (1-click signup, autocomplete). Time prompts correctly. Surface them when motivation is highest.
Cialdini's Principles
Persuasion
Social proof, scarcity, authority, reciprocity, commitment, and liking — the 6 principles of influence.
→ Show user counts, testimonials, activity feeds. Countdown for limited offers. Authority badges and certs.
Design & Architecture
Progressive Disclosure
Complexity Management
Show only the information and controls needed at each step. Reveal complexity progressively as the user advances.
→ Wizard flows for complex tasks. "Advanced settings" accordions. Show more / expand patterns.
Recognition over Recall
Cognitive Load
Minimise memory load by making objects, actions, and options visible. Recognition is far easier than recall.
→ Auto-complete, recently used items, visible shortcuts, descriptive labels. Never require memorised commands.
Tesler's Law
Complexity Conservation
Every system has a certain amount of irreducible complexity. The only question is who handles it — user or designer.
→ Smart defaults. Auto-save. Backend handles complexity so UI stays simple. "Just works" philosophy.
Aesthetic-Usability Effect
Perceived Quality
Users perceive aesthetically pleasing designs as more usable. Beautiful UI is perceived as better-functioning UI.
→ Invest in typography, spacing, and motion. Polish empty states. Consistent visual language throughout.
Error Prevention
Nielsen Heuristic
Prevent problems from occurring in the first place. Confirmations, constraints, and good defaults eliminate errors.
→ Inline validation before submit. Confirm destructive actions. Disable invalid states. Undo over delete.
Consistency & Standards
Nielsen Heuristic
Users shouldn't have to wonder whether different words, situations, or actions mean the same thing.
→ Design tokens for all visual values. Shared component library. Consistent terminology across all copy.
Accessibility & Inclusion
WCAG 2.2 AA
Accessibility Standard
Web Content Accessibility Guidelines — the legal and ethical baseline for inclusive digital experiences.
→ 4.5:1 contrast ratio, keyboard navigable, ARIA labels, focus visible, no seizure-trigger content.
Inclusive Design
Microsoft Principles
Design for the extremes and it will work for the mainstream. Solve for one, extend to many.
→ Captions for video, large-text mode, high-contrast theme, touch targets 44px+, reduced-motion support.
