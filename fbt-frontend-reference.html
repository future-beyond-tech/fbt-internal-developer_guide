FRONTEND PARADIGMS

mental models
⚛️
Declarative UI
REACT'S CORE

Describe what to render, not how to manipulate DOM. React reconciles virtual DOM diff to real DOM. State changes trigger re-renders automatically. Mental model: UI = f(state).

ReactVirtual DOM
🔄
Functional Components
HOOKS ERA

Pure functions that take props, return JSX. Hooks for side effects and state. Closures capture state. Avoid class components — they carry baggage. Easier to test, compose, and reason about.

React 16.8+Hooks
🏝️
Islands Architecture
PARTIAL HYDRATION

Most of page is static HTML. Only interactive islands hydrate with JS. Astro pioneered this. Next.js Server Components approximate it. Result: dramatically less JS shipped.

Next.jsAstro
🔀
Streaming SSR
REACT 18+

Stream HTML chunks to browser as they're ready. Suspense boundaries control what streams when. Critical content first, slower parts stream in. LCP improves dramatically.

React 18Next.js 15
🧱
Atomic Design
BRAD FROST

Atoms → Molecules → Organisms → Templates → Pages. Bottom-up component hierarchy. Each level adds context. Atoms are elemental (Button, Input). Pages assemble everything.

Design SystemComponent Lib
📦
Micro-Frontends
DISTRIBUTED UI

Multiple teams own independent frontend slices. Module Federation (Webpack), iframes, or Web Components as integration layers. High overhead — only at org scale.

Custom Hooks
LOGIC EXTRACTION

Extract stateful logic into reusable functions. Start with use prefix. Can compose other hooks. Replace HOCs and render props in modern React. Single most powerful abstraction.

function useDebounce<T>(value: T, delay: number) { const [dv, setDv] = useState(value) useEffect(() => { const t = setTimeout(() => setDv(value), delay) return () => clearTimeout(t) }, [value, delay]) return dv }
TSX

🎭
Render Props
FUNCTION-AS-CHILD

Pass a function as a prop that returns JSX. Enables sharing stateful behavior without HOC complexity. Now mostly superseded by hooks, but still useful for complex render delegation (tables, virtual lists).

<DataLoader url="/api/users" render={(data, loading) => loading ? <Spinner/> : <UserList data={data}/> } />
TSX
Legacy patternUse hooks instead
🌊
Controlled vs Uncontrolled
STATE OWNERSHIP

Controlled: Parent owns state, passes value + onChange. Fully predictable, testable. Uncontrolled: DOM owns state, read via ref. Less re-renders. React Hook Form uses uncontrolled internally for performance.

Controlled = full powerUncontrolled = perf
🔀
Higher-Order Components
HOC PATTERN

Function that takes a component, returns enhanced component. withAuth(), withTheme(). Mostly superseded by hooks. Still valid for class components and some cross-cutting concerns (code splitting).

LOGIC WITHOUT STYLE

Provide behavior and accessibility with zero styling opinions. Radix UI, Headless UI, React Aria. You bring the styles. Maximum flexibility for design systems. Industry best practice for component libraries.

FRONTEND DESIGN PATTERNS

solutions to recurring problems
⚛️
React-Specific Patterns
Provider Pattern
CONTEXT
Wrap subtree with context provider. Children consume via useContext. ThemeProvider, AuthProvider, QueryClientProvider. Avoid mega-provider — split concerns.
Global/shared state injection
Container/Presenter
SEPARATION
Container fetches data and manages state. Presenter receives props, renders UI. Clean testing — test presenter with mocked props, container logic separately.
Complex data + UI separation
Skeleton Loading
UX PATTERN
Render placeholder shapes while loading. Reduces perceived wait time vs spinner. Match content layout exactly. React Suspense + skeleton component.
Every async data component
Optimistic Updates
UX / MUTATION
Update UI immediately, sync to server in background. Roll back on error. React Query useMutation onMutate/onError/onSettled. Instant feel for like, follow, delete.
Low-risk mutations (likes, toggles)
Error Boundary
FAULT ISOLATION
Catch JS errors in subtree, render fallback UI. Class component (React 18) or react-error-boundary library. Every page should have one. Granular = better UX.
All production page routes
Suspense Boundary
ASYNC LOADING
Declaratively handle async rendering. Show fallback while children are loading. Compose multiple Suspense boundaries for granular loading states. Works with use() hook in React 19.
Async data, lazy-loaded components
Slot Pattern
COMPOSITION
Named slots for flexible layout composition. {header, children, footer} props or named children slots. Modal, Card, Layout components use this heavily.
Layout components with zones
Virtualization
PERFORMANCE
Only render visible items in long lists. React Virtual (TanStack), react-window. 10,000-item list performs like a 20-item list. Essential for data grids, infinite scroll.
Lists >100 items
Portals
DOM ESCAPE
Render children outside parent DOM tree. Modals, tooltips, dropdowns that need to escape z-index/overflow constraints. createPortal(children, domNode).
Modals, overlays, toasts
Derivation over Storage
STATE MANAGEMENT
Don't store derived state — compute it. filteredItems = useMemo(() => items.filter...). Avoid storing selectedItemDetails if you can look it up from selectedId + items array.
Anytime you're tempted to sync state

Caching Model (Next.js 15)
CHANGED IN v15

Next.js 15 reversal: fetch() requests are now uncached by default. Opt-in to caching explicitly. GET Route Handlers are also uncached by default. Breaking change from v14.

// No cache (default in v15) const data = await fetch(url) // Cached permanently (opt-in) const data = await fetch(url, { next: { revalidate: false } }) // Cache with time (ISR-like) const data = await fetch(url, { next: { revalidate: 3600 } }) // On-demand revalidation revalidatePath('/products') revalidateTag('product-list')

Async Params (Next.js 15)
BREAKING CHANGE

params and searchParams are now async Promises in Next.js 15. Must await them. Use React.use() for client components or await in server components.

RSC & SERVER ACTIONS

zero-bundle server logic
🔒
Server Components (RSC)
DEFAULT IN APP ROUTER

Run on server only. Zero client-side JS. Direct DB access, use secrets. Can be async. Cannot use useState, useEffect, event handlers, or browser APIs.

// server-component.tsx (no directive needed) async function UserProfile({ id }: { id: string }) { // Direct DB access — no API needed const user = await db.user.findUnique({ where: { id } }) return <div>{user?.name}div> }
RSC
0kb JS bundleDB directSecrets safe
💻
Client Components
'USE CLIENT' DIRECTIVE

Add 'use client' at top of file. Can use hooks, event handlers, browser APIs. Still server-rendered HTML for initial paint (SSR), then hydrated. Keep client components as leaf nodes.

'use client' import { useState } from 'react' export function Counter() { const [count, setCount] = useState(0) return <button onClick={() => setCount(c => c+1)}> {count} button> }
TSX
Has bundle costPush to leaves
⚡
Server Actions
FULL-STACK MUTATIONS

Async functions that run on server, called from client. Replace REST endpoints for mutations. Progressive enhancement — work without JS. Integrated with React forms, useTransition, React 19 useActionState.

'use server' // can be at top of file or inline export async function createUser(data: FormData) { const parsed = UserSchema.safeParse( Object.fromEntries(data) ) if (!parsed.success) return { error: parsed.error } await db.user.create({ data: parsed.data }) revalidatePath('/users') }
Server Action
Progressive EnhancementuseActionState

RSC Composition Rules
CRITICAL MENTAL MODEL

Server Components can import and render Client Components. Client Components cannot import Server Components (but can receive them as children/props). Pass SC as children to CC for island architecture.

// ✅ Pass RSC as children to Client <ClientWrapper> <ServerFeed /> // passed as children ClientWrapper> // ❌ Cannot import RSC inside CC // import ServerFeed inside 'use client' = error

API INTEGRATION PATTERNS

🏗️
API Client Layer
ABSTRACTION PATTERN

Centralize all API calls in a typed client layer. Never call fetch() directly from components. Enables easy mocking, error handling, auth token injection, and type safety end-to-end.

// lib/api/users.ts export const usersApi = { list: () => apiClient.get<User[]>('/users'), byId: (id: string) => apiClient.get<User>(`/users/${id}`), create: (data: CreateUserInput) => apiClient.post<User>('/users', data), }
TS
📡
Loading / Error / Empty States
ALWAYS ALL THREE

Every async component must handle all three states. Never render undefined data. Use React Query's isLoading, isError, data states. Skeleton for loading, error boundary for errors, empty state illustration for zero results.

const { data, isLoading, error } = useUsers() if (isLoading) return <UserListSkeleton/> if (error) return <ErrorState error={error}/> if (!data?.length) return <EmptyState/> return <UserList users={data}/>
TSX
🔄
Infinite Scroll / Pagination
TANSTACK QUERY

useInfiniteQuery for infinite scroll. Cursor-based pagination. Intersection Observer triggers next page. React Query handles all page caching, prefetching, and state merging automatically.

const { data, fetchNextPage, hasNextPage } = useInfiniteQuery({ queryKey: ['posts'], queryFn: ({ pageParam }) => fetchPosts(pageParam), getNextPageParam: (last) => last.nextCursor, initialPageParam: undefined })
