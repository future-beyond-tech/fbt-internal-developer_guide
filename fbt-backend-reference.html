CQRS & Event Sourcing

Core FBT Pattern
↕️
CQRS
COMMAND QUERY RESPONSIBILITY SEGREGATION

What: Separate the write model (Commands) from the read model (Queries). Each is optimized independently.

How: Commands mutate state, return void or ID. Queries are pure reads, return DTOs. Two separate stacks.

Why: Read and write workloads are different. Independent scaling, simplified models.

When: Complex domains, high read/write ratio imbalance, event sourcing.

// Command
public record CreateBookingCommand(Guid PropertyId, DateOnly Check);
// Query
public record GetBookingQuery(Guid BookingId);
// Handler via MediatR
public class CreateBookingHandler : IRequestHandler<CreateBookingCommand>
C#
.NET (MediatR)PythonRust
📚
Event Sourcing
IMMUTABLE AUDIT LOG

What: Store state as a sequence of events, not current state snapshot.

How: Append events to event store. Rebuild state by replaying. EventStore DB, Marten in .NET, EventStoreDB.

Why: Complete audit log, temporal queries, event replay, natural fit for CQRS.

When: Financial systems, booking, compliance-heavy domains.

// Events are immutable facts
BookingCreated, BookingCancelled, PaymentReceived

// Rebuild aggregate from events
aggregate = events.Aggregate(initial, apply)
Concept
.NETPython
🗺️
CQRS + ES Full Flow
1
HTTP Request → Controller
Thin controller validates HTTP contract, maps to Command/Query object, dispatches via MediatR.
2
Pipeline Behaviors (Middleware)
Logging → Validation (FluentValidation) → Authorization → Caching → Performance → Handler.
3
Command Handler → Aggregate
Load aggregate from repo. Call domain method. Aggregate raises DomainEvent(s). Save to store.
4
Domain Events → Outbox
DomainEvents saved in Outbox table in same transaction. Background processor publishes to message broker.
5
Projections Update Read Models
Event consumers rebuild denormalized read models (Redis, separate DB tables, Elasticsearch).
6
Query Handler → Read Model
Queries bypass write model entirely. Hit denormalized read store directly. Return DTOs.

Performance Optimization

🗂️
Caching Strategy

L1: In-memory (IMemoryCache). L2: Redis (IDistributedCache). L3: CDN edge. Cache-aside is default. Set appropriate TTL. Invalidate on write.

ALL
🔢
Pagination

Never return unbounded result sets. Cursor-based for large/live data. Offset for admin panels. Page size max 100. EF Core Skip/Take. SQLAlchemy paginate.

ALL
📦
Batch Processing

Process records in bulk, not one by one. SaveChangesAsync once after batch inserts. Python list comprehensions over loops. Rust iterator chains are zero-cost.

ALL
🗺️
Database Indexing

Index every FK, query filter field, sort field. Composite indexes for multi-column filters. EXPLAIN ANALYZE every slow query. Avoid full table scans.

ALL
⚡

Never block async with .Result or .Wait(). Use ValueTask for hot paths in .NET. Python asyncio gather for concurrent I/O. Tokio join! for concurrent futures in Rust.

ALL
🏊

.NET / C# Specific Techniques

💉
Dependency Injection
BUILT-IN DI CONTAINER

AddSingleton, AddScoped, AddTransient lifetimes. Avoid service locator anti-pattern. Keyed services (.NET 8). Factory registration for complex scenarios.

🔗
Middleware Pipeline
ASP.NET CORE

app.Use() for middleware. Order matters. Exception → Auth → CORS → Rate Limit → Routing → Controllers. Custom middleware for correlation IDs, tenancy resolution.

📋
FluentValidation
VALIDATION PIPELINE

Decouple validation from models. AbstractValidator<T>. Integrate with MediatR pipeline behaviors. Validate at application boundary. Domain validation stays in aggregates.

🗺️
AutoMapper / Mapster
OBJECT MAPPING

Map domain → DTO automatically. Explicit mappings over convention magic. Mapster is faster than AutoMapper. For Rust: use From/Into traits manually — no magic.

⚙️
Background Services
IHOSTEDSERVICE

IHostedService / BackgroundService for long-running tasks. Outbox publisher, event consumer, health watchdog. Combine with Channels for producer-consumer patterns.

🏗️
Vertical Slice Architecture
FEATURE FOLDERS

Organize by feature, not layer. Each slice has its own Command/Query/Validator/Handler/Response. Jimmy Bogard pattern. Excellent for CQRS with MediatR. Less cross-layer coordination.

Python Specific Techniques

🚀
FastAPI
ASYNC WEB FRAMEWORK

Pydantic models for request/response validation. Dependency injection via Depends(). Background tasks. OpenAPI auto-generated. Lifespan events for startup/shutdown.

📦
Pydantic v2
DATA VALIDATION

Runtime type validation. BaseModel for all DTOs and domain models. Settings management with BaseSettings. 5-50x faster than v1. Rust-backed core.

🗄️
SQLAlchemy 2.x
ASYNC ORM

Async engine + session. DeclarativeBase. Alembic for migrations. Use select() syntax (2.x style). asyncpg driver for PostgreSQL. Avoid lazy loading in async context.

🎯
Dependency Injection
DEPENDENCY-INJECTOR

dependency-injector library. Wiring to FastAPI Depends(). Containers for service registration. Providers: Factory, Singleton, Scoped. No service locator.

🔄
Async Patterns
ASYNCIO MASTERY

asyncio.gather() for concurrent I/O. asyncio.TaskGroup (3.11+). anyio for backend-agnostic async. Never mix sync I/O in async functions. Use run_in_executor for CPU work.

🧠
Dataclasses / Protocols
STRUCTURAL TYPING

@dataclass for value objects. @dataclass(frozen=True) for immutability. Protocol over ABC for structural duck typing. TypedDict for typed dicts. Never use bare dict for domain data.

Type Safety Stack
mypy strict mode + pyright. No untyped functions. reveal_type() for debugging. beartype for runtime checks.
Package Management
uv (fastest), Poetry, or pip-tools. pyproject.toml only. Virtual envs always. Lockfile committed.
Testing Stack
pytest + pytest-asyncio + httpx + factory-boy + Hypothesis + testcontainers-python.
ML/AI Integration
FastAPI + Pydantic + torch/transformers on Modal GPU. Async inference endpoints. Qdrant for vector store.

Rust Specific Techniques

⚡
Tokio Async Runtime
ASYNC FOUNDATION

tokio::main macro. Spawn tasks with tokio::spawn(). join! for concurrent futures. select! for racing. tokio::time::sleep. sync primitives: Mutex, RwLock, Semaphore, watch, mpsc, broadcast channels.

🌐
Axum Web Framework
TOWER-BASED HTTP

Router + Handler + Extractor pattern. State injected via Extension/State. Tower middleware for auth, logging, rate limiting. Axum macros for routing. Type-safe extractors.

🗄️
sqlx + SeaORM
DATABASE ACCESS

sqlx: compile-time checked SQL queries. SeaORM: async active record style ORM. sqlx::query_as! macro maps to structs. Migrations via sqlx-cli or sea-orm-migration.

🔧
Error Handling
RESULT-FIRST

thiserror for library errors (derive macro). anyhow for application errors (box dyn). ? operator for propagation. Never unwrap() in production. Custom error enums for domain errors.

🧩
Trait-Based Design
RUST INTERFACES

Traits as interfaces. Trait objects (dyn Trait) for runtime polymorphism. Generics + trait bounds for compile-time dispatch (zero-cost). Newtype pattern for type safety without inheritance.

📦
Serde Serialization
ZERO-COST SERIALIZATION

Derive Serialize/Deserialize on all data types. Custom field names via #[serde(rename)]. Skip fields with #[serde(skip)]. Flatten nested structs. JSON, TOML, YAML, MessagePack.

🦀
Rust for Systems: Use Rust for performance-critical services (API gateway, data pipeline, media processing), security-critical code (crypto, auth validators), and anything that must not crash. Vyxnos Shield is Rust-first for zero-trust gateway logic.
